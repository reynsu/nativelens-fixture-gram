# Detox + nativelens

Detox jest setup pre-wired with `@reynsu/nativelens-detox-adapter`. The adapter is what tags each end-of-action snapshot with the correct `stepId` so the harness can produce per-step diffs.

## Layout

- `.detoxrc.js` (repo root) — Detox config generated by `npx detox init -r jest`.
- `e2e/jest.config.js` — Jest config for Detox specs. Sets `setupFilesAfterEnv` to `jest.setup.ts`.
- `e2e/jest.setup.ts` — Wires the nativelens adapter. Exposes `step(stepId, fn)` as a global.
- `babel.config.js` (repo root) — `babel-preset-expo` so Jest can transform the `.ts` setup file. Matches Expo's internal babel default; safe for the app runtime.

## What's missing before tests can actually run

The `.detoxrc.js` generated by Detox has `YOUR_APP` placeholders. Before Detox can launch the iOS build:

```bash
npx expo prebuild --platform ios   # generates ios/ folder + xcworkspace
# then edit .detoxrc.js: replace YOUR_APP with the actual scheme name (likely "nativelensfixturegram")
detox build -c ios.sim.debug
detox test -c ios.sim.debug
```

## Writing specs

`step` is exposed globally by `jest.setup.ts`. Use it to wrap any Detox action you want to capture as a discrete snapshot:

```ts
describe("Pixly smoke", () => {
  beforeAll(async () => { await device.launchApp(); });

  it("logs in to feed", async () => {
    await step("tap-submit", async () => {
      await element(by.id("login-submit")).tap();
    });
    await step("verify-feed", async () => {
      await expect(element(by.id("feed-screen"))).toBeVisible();
    });
  });
});
```

The adapter sends `step:set` → waits for probe's `step:ack` → runs the action → captures the snapshot tagged with the stepId.

## With the harness running

```bash
nativelens run        # starts harness on :7878, dashboard on :7777
detox test -c ios.sim.debug
```

Override harness URL with `NATIVELENS_HARNESS_URL=ws://other-host:7878 detox test ...`.
